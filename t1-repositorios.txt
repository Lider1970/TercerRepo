GIT (basado en pildoras informáticas)
Sofware de control de versiones.

TEMA 1 CREANDO REPOSITORIOS
1.--------------- git init
Crea un directorio de trabajo.
Para empezar, nos ubicamos en la carpeta donde desarrollaremos el proyecto y desde ahí abrimos git bash. Este comando se ejecuta una vez, al comenzar un proyecto, creando una carpeta oculta llamada ".git".
Crea dos areas: area de ensayo (staging area) y repositorio local.

2. ------ git status -s
Este comando nos muestra todos los documentos que existen en esa carpeta, antecedido por signos o letras.
Nos postrará dos signos de interrogación, si no esta siendo monitoreado por git (está solo en nuestro area de trabajo).
Nos mostrará una "A" cuando lo hubiesemos subido a stating area.
Esto significa que este comendo sólo nos muestra aquellos documentos libre de seguimiento y aquellos subidos al stating area.

3.------- git add <file>
Esto nos permite dar inicio al seguimiento de una o varias carpetas, porque lo adiciona al area de ensayo.

4.-------- git commit
Esto nos permite tomar una foto una instantanea de un determinado documento. Por ejemplo ponemos: git commit -m "Este es mi primer commit"
Al ejecutar git nos dirá las lineas cambiadas y el total de lineas del documento, y además creará como objeto el documento.
Si aquí usamos git status -s, veremos que ya no nos muestra el archivo que comiteamos.

5.----- PROBAMOS A CAMBIAR EL ARCHIVO COMITEADO
Hacemos cambios al documento comiteado y después usamos git status -s, como resultado tendremos que ahora el archivo index.html está antecedido por un espacio y una letra M en rojo, de esta forma git me informa que ese archivo no esta en stating area, ha sido modificado y que no esta respaldado por git.
Para hacer commit del documento con dos modificaciones, primero debo usar git add, y después git commit -m.
Con git status -s, puedo verificar que el archivo index.html adicionado esta en el stating area, porque me muestra antecedido por la letra M en verde y un espacio.
Ahora ya puedo comitearlo, por ejemplo: git commit -m "Dos modificaciones: título y primer párrafo", y me dira que fue comiteado.

6.--- git log --oneline
Con este comando puedo ver el listado de todos los commit realizados, con su respectiva ubicación, identificación y descripción.

7.----- git reset --hard <file codigo>
De esta forma borramos y nos ubicamos en el commit que decidamos, que elegimos usando el código asignado al commit. En este caso, se borraron los dos cambios que había realizado y ese borrado es irrecuperable.
